1.6 Clarity requires detail DESCRIPTION To provide access to information at the highest resolution available to the organization, concepts must be discrete in the model. For example, “net income” contains two concepts which each need to be defined and managed independently within a model: the concept of “income” - an observed value, and the concept of “net” - a derived value comprising the sum of income and expenses. Combining concepts this way is called chaining and should only occur when each element is also separately addressed. Similarly, overloading of meaning is not acceptable – i.e. it is unacceptable to assign two discrete concepts to one element or term. RATIONALE • Combining concepts hides meaning and reduces reuse. • “Chaining” prevents independent governance of component concepts. IMPLICATION • Break down compound concepts into atomic entities/attributes. • Avoid semantic overloading of attributes. • Ensure derived values reference explicit underlying components.
