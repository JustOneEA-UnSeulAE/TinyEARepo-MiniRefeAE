2.7 An Enterprise Data Reference Model (EDRM) is a set of reusable data requirements. The EDRM comprises the conceptual and logical data models, conceptual business process model, business glossary, and data related business rules. The complete EDRM is an enterprise level statement of requirements pertaining to data. All solution design should comply with the reference model, at minimum from an interoperability perspective, and ideally also from a data-at-rest perspective. Rationale • Enterprise consistency enables interoperability and reuse. • Solutions must align with shared data foundations. Architecture Implications • All solutions must demonstrate EDRM compliance. • EDRM governs both integration and data-at-rest designs. • Treat the EDRM as an enterprise contract, not a reference only.
