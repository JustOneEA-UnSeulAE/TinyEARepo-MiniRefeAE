1.13 Four common attributes DESCRIPTION Many entities share some or all of the following four common attributes that should be consistently identifiable and defined across the EDRM: the date the entity came into existence, the date the entity ceased to exist, the name of the entity, and the type of the entity. RATIONALE • Many entities share lifecycle and identification characteristics. • Consistency simplifies governance and integration. IMPLICATION • Standardize definitions for: o Start date o End date o Name o Type • Apply consistently across the EDRM. • Use enterprise-wide standards for these attributes.
