1.9 An entity plays a role in each association DESCRIPTION The attributes of the association include attributes of the roles played by the associated entities: these must not be confused with attributes of the entity itself. For example, an employee number is an attribute of the association between a person in the role of employee, and an organization in the role of employer. This is evident because a person may have more than one concurrent employee number if they have more than one concurrent employment. Conversely, the legal name of an employee is an attribute of the entity playing the role, the person, because there is only one, even when that person engages in multiple employments. Note however, that if it is desired to record the name given by an employee to an employer, it may be necessary to record an employment name as well. Care should always be taken to ensure that roles are not confused with the underlying entities. RATIONALE • Attributes often belong to a relationship, not the entity itself. • Confusing roles with entities causes data duplication and error. IMPLICATION • Model role attributes on associations, not entities. • Explicitly name and define roles in associations. • Validate that entity attributes remain invariant across roles.
