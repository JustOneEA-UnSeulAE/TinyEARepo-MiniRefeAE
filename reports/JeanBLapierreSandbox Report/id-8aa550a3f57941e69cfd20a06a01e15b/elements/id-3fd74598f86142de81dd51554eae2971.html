2.5 A business glossary defines terminology. A business glossary defines all terminology used in the EDRM. It also defines any acronyms and abbreviations. For consistency, terminology is usually expressed on models in the singular form, in present tense. Rationale • Shared language is foundational to shared data. • Inconsistent terms undermine enterprise models. Architecture Implications • All model terms must reference glossary entries. • Enforce singular, present-tense naming conventions. • Govern glossary changes alongside model changes.
