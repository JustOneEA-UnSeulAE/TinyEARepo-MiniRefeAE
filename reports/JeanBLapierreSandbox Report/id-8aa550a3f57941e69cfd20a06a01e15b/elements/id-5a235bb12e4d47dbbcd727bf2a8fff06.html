A1.08.06 Data Profiling [DACM] ()
A1.08.06 Data Profiling Software that supports all forms of data analysis of extremely large, complex data sets (big data) that are manipulated for business consumption. Data profiling is the process of examining the data available in an existing data source (e.g. a database or a file) and collecting statistics and information about that data. The purpose of these statistics may be to: - Find out whether existing data can easily be used for other purposes. - Improve the ability to search the data by tagging it with keywords, descriptions, or assigning it to a category. - Give metrics on data quality including whether the data conforms to particular standards or patterns. - Assess the risk involved in integrating data for new applications, including the challenges of joins. - Assess whether metadata accurately describes the actual values in the source database. - Understanding data challenges early in any data intensive project, so that late project surprises are avoided. Finding data problems late in the project can lead to delays and cost overruns. - Have an enterprise view of all data, for uses such as master data management where key data is needed, or data governance for improving data quality. A1.08.06 Profilage de données Logiciel qui prend en charge toutes les formes d’analyse de données d’ensembles de données extrêmement vastes et complexes (mégadonnées), qui sont manipulés à des fins de consommation par les entreprises.Le profilage de données est le processus qui consiste à examiner les données disponibles dans une source de données existante (p. ex., une base de données ou un fichier) et à recueillir des statistiques et des renseignements sur ces données. Ces statistiques peuvent avoir pour but :- de déterminer si les données existantes peuvent facilement être utilisées à d’autres fins;- d’améliorer les capacités de recherche de données marquant celles-ci avec des mots clés ou des descriptions, ou en les assignant à une catégorie;- de fournir des mesures sur la qualité des données, y compris si les données sont conformes à des normes ou des modèles particuliers;- d’évaluer le risque associé à l’intégration des données pour les nouvelles applications, y compris les défis liés aux données pivots;- d’évaluer si les métadonnées décrivent avec exactitude les valeurs réelles dans la base de données source;- de comprendre les défis liés aux données dès le début de tout projet à forte intensité en matière de données, afin d’éviter les surprises tardives dans le cadre des projets. Le fait de trouver des problèmes en matière de données tardivement dans le cadre d’un projet peut entraîner des retards et des dépassements de coûts;- d’avoir une vue d’ensemble de toutes les données, pour des utilisations comme la gestion des données de base lorsque des données clés sont nécessaires, ou la gouvernance des données pour améliorer la qualité des données.
