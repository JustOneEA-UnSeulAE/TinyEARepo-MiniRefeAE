1.12 Entities have types, super types and subtypes DESCRIPTION Most entities are sufficiently complex to be categorized in several different ways. The primary ways that entities can be distinguished into groups is usually given the label type. For example, a husky is a type of dog, and a dog is a type of mammal. If we arbitrarily define an entity called dog, the super type will be mammal, and one of the subtypes will be husky. Alternatively, we could define the entity to be mammal, with a super type of animal, a subtype of dog, and a sub-subtype of husky. The reference point choice between dog or mammal is arbitrary, and unimportant. RATIONALE • Classification supports reuse and abstraction. • Reference point selection is arbitrary but necessary. IMPLICATION • Use subtype hierarchies to manage complexity. • Avoid duplicating attributes across related entities. • Ensure subtype rules are explicit and governable.
