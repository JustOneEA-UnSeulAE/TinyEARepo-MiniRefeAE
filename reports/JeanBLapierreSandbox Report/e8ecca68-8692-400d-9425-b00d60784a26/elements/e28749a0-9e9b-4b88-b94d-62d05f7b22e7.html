Define an SLO up Front ()
each API should have a clearly defined Service Level Objective (SLO), which should include: o Support contact and availability o Service uptime objective (e.g., 99%) o Support response time (e.g., within the hour, 24 hours, best effort) o Scheduled outages (e.g., nightly, weekly, every 2nd Sunday evening) o Throughput limit (e.g., 100 requests per second per consumer) o Message size limit (e.g., <1Mb per request)
