Guideline A.3.7: (For the authoring tool user interface) Ensure that previews are at least as accessible as in-market user agents. [Implementing A.3.7] Rationale: Preview features are provided by many authoring tools because the workflow of authors often includes periodically checking how user agents will display the web content to end users. Authors with disabilities need the same opportunity to check their work. A.3.7.1 Preview (Minimum): If a preview is provided, then at least one of the following is true: (Level A) (a) In-Market User Agent: The preview renders content using a user agent that is in-market; or (b) UAAG (Level A): The preview conforms to the User Agent Accessibility Guidelines 1.0 Level A [UAAG10].
